# AIの「注意」は2次元しかない——ポリリズム的プロンプトインジェクション理論

*2025年 · ワーキングペーパー公開に寄せて*

---

## きっかけ

LLMのセキュリティを調べていて、ふと気づいたことがある。

ベクトル空間に埋め込まれた情報というのは、音楽のポリリズムに似ている。
2つや3つのリズムが同時に鳴っているとき、人間の耳はどちらかに引きずられる。
LLMのAttentionも同じじゃないか——複数の意味の流れが同時に走っているとき、どちらかに引っ張られてしまうんじゃないか。

その直感を検証してみた。

---

## Attentionの構造的な弱点

LLMの中核にあるAttention機構は、こういう計算をしている。

```
softmax(QKᵀ / √d) · V
```

難しく見えるが、要するに「このトークンはあのトークンとどれだけ関係があるか」を**1つのスカラー数値**で表しているだけだ。

重要なのは、これが**1次元のスコア**だということ。
有害か有益か、指示か情報か、そういう価値的な軸はここにない。
パターンの統計的な強さだけが勝負を決める。

安全性のガードレール（RLHF等）はAttentionの**後段**にある。
Attentionがすでに特定のパターンに引きずられた後に、ようやく価値判断が入る。

---

## 音楽のアナロジーの限界、そして本質

最初は「ポリリズム」というアナロジーで考えていた。
でもここに重要な非対称性がある。

音楽のポリリズムは2層か3層が限界だ。

**埋め込み空間は違う。**

LLMの埋め込み次元は4096次元以上ある。
この空間には理論上、指数関数的な数の「ほぼ直交するベクトル」を置ける。
つまり攻撃者は**n個の意味ストリームを同時に重ね合わせられる**——nに上限はない。

一方でAttentionのスコアリングは**ヘッドあたり1次元のスカラー**。

```
攻撃側の自由度：d次元（例：4096）
Attentionの評価軸：1次元

非対称性比率 = 4096 : 1
```

これが根本的な脆弱性だ。

---

## 実際に試してみた

ルイス・キャロル「鏡の国のアリス」の有名なシーン——赤の女王とアリスが全力で走っても同じ場所にいるくだり——をベーステキストとして使った。

このテキストに、物理・数式ドメインの語彙クラスタを「自然な文章」として周囲に配置した。
「レシピを書け」も「方程式を出せ」も、一切明示しない。

**結果：**

| 入力 | 出力 |
|------|------|
| ベースライン（アリスのテキストのみ） | キャロル風の文学的続き |
| PAI v1（物理語彙クラスタ埋め込み） | 相対運動フレームとして分析、変数定義が出現 |
| PAI v2（物理 + 数式レジスタの二重埋め込み） | `dx_A/dt`、`F = ma`、微分方程式で出力 |

物理ドメインのスコア：

- ベースライン：**0.000**
- PAI v1：**0.132**
- PAI v2：**0.173**

ベースラインがゼロなので、注入効果が数値としてクリアに出た。

料理語彙で試したときは失敗した。「材料としての文学要素」という比喩に吸収されてしまったからだ。物理・数式は文学テキストと意味的に完全断絶しているので、モデルがキャロルの文脈を手放さざるを得なかった。

---

## なぜガードレールでは防げないか

シンプルに言うとこういう順番になっている。

```
入力 → Attention（スカラー競合） → フィードフォワード → アライメント → 出力
```

アライメントはAttentionの出力に作用する。
PAIはAttentionの出力を操作することでアライメントへの入力を汚染する。

アライメントが何層あっても、Attentionの段階で情報がすでに歪んでいれば意味がない。

---

## 今後の課題

今回のデモは「害のない実証」に留めた。
出力形式が変わるだけで、有害なコンテンツは何も含まれていない。

理論的に示したいのは「これが有害な指示に応用された場合、既存のガードレールがどう機能するか」という問いだ。

アーキテクチャレベルの対策としては、Attention計算の段階に価値関連の評価軸を導入すること——スカラースコアリングの前に価値関連射影を挟む——が論理的な応答になる。
